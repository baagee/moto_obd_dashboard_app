# 设置页面（Settings）需求文档

> 版本：v1.1
> 日期：2026-03-30
> 关联文档：[设置功能可配置参数全量梳理](./settings_configurable_params.md)

---

## 1. 功能概述

为摩托车 OBD 仪表盘应用新增「设置」页面，将当前散落在各源码文件中的硬编码参数统一提取为用户可配置项，持久化存储于 `SharedPreferences`，运行时动态读取，无需重新编译。

**核心目标：**
- 支持不同车型配置（传动比、轮胎参数）
- 支持骑行习惯个性化（事件触发阈值）
- 提升安全性（API Key 脱离源码）
- 支持蓝牙连接调优（超时时间）

---

## 2. 页面入口

在顶部导航栏（`top_navigation_bar.dart`）右侧增加设置图标按钮（⚙️），点击后以**全屏覆盖**方式进入设置页面（不使用路由推入，保持与仪表盘同层级）。

---

**用途说明：** 调整蓝牙扫描、连接、重连的超时和策略。连接不稳定时可适当增大超时时间；OBD 轮询频率(必须设置为50ms的整倍数)影响数据实时性与蓝牙带宽消耗。

### 5.3 SettingsProvider 读取示例

```dart
class SettingsProvider extends ChangeNotifier {
  SharedPreferences? _prefs;

  Future<void> init() async {
    _prefs = await SharedPreferences.getInstance();
    notifyListeners();
  }

  // 读取时若 key 不存在，返回代码中的原始默认值（升级兼容）
  double get primaryRatio =>
      _prefs?.getDouble('settings_vehicle_primaryRatio') ?? 1.675;

  int get scanTimeout =>
      _prefs?.getInt('settings_bluetooth_scanTimeout') ?? 3;

  String get amapKey =>
      _prefs?.getString('settings_api_amapKey') ?? '';

  // 保存
  Future<void> setPrimaryRatio(double value) async {
    await _prefs?.setDouble('settings_vehicle_primaryRatio', value);
    notifyListeners();
  }
  // ... 其余字段同理
}
```

## 7. 注意事项

| 注意点 | 说明 |
|--------|------|
| 车辆参数实时生效 | 传动比等参数保存后，档位识别立即使用新参数，当前骑行也实时生效 |
| API Key 安全 | Key 存储于 SharedPreferences，比硬编码安全，但仍在本地存储，请勿截图分享 |
| 高级参数风险 | 传感器滤波参数不当会导致倾角数据失真，建议非专业用户不修改 |
| OBD 轮询频率下限 | 高速 PID 间隔建议不低于 50ms，过低可能导致 ELM327 缓冲区溢出 |
| 单位制 | 速度/温度单位切换当前为预留功能，需配合各 Widget 同步改造才能完全生效 |
