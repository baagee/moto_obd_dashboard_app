# 档位识别算法详解

## 概述

本文档详细解释档位识别算法的核心原理、数据流、优化策略和参数调优方法。

---

## 一、核心原理

### 1.1 基础公式

通过**转速**和**车速**反推当前档位：

```
实时传动比 = (转速 × 60 × 轮胎周长) / (速度 × 1000)
```

**物理含义**：
- 分子：发动机转一圈，轮胎转的距离（米/分钟）
- 分母：车辆行驶距离（米/分钟）
- 结果：发动机转多少圈，轮胎转一圈

### 1.2 档位匹配原理

每个档位有固定的**理论传动比**：

```
理论总传动比 = 初级减速比 × 该档齿轮比 × 最终减速比
```

将实时计算的传动比与各档理论值比对，找出最接近的档位。

## 三、核心算法详解

### 3.1 Log 误差计算

**为什么用 Log 误差？**

线性误差：`|测量值 - 理论值| / 理论值`

- 1档传动比 14，误差 1 → 相对误差 7%
- 6档传动比 4.4，误差 1 → 相对误差 22%

问题：同样的绝对误差，在高档位影响更大！

Log 误差：`|log(测量值) - log(理论值)|`

- 本质：对数空间的距离
- 物理意义：比例误差
- 优势：对高档位比例误差更敏感，识别更准确

### 3.2 动态阈值

**问题**：低档位传动比差距大（14→10→7.5），高档位差距小（5.96→5.03→4.44）

**解决方案**：动态阈值公式

```
阈值 = 0.08 + 0.8 / 理论传动比
```

| 档位 | 理论传动比 | 阈值 |
|------|-----------|------|
| 1档 | 14.27 | 0.14 (14%) |
| 2档 | 10.21 | 0.16 (16%) |
| 3档 | 7.90 | 0.18 (18%) |
| 4档 | 6.58 | 0.20 (20%) |
| 5档 | 5.71 | 0.22 (22%) |
| 6档 | 5.14 | 0.24 (24%) |

**效果**：低档严格，高档宽松

### 3.3 换挡瞬态处理

**问题**：
- 升档时：rpm 瞬间下降，但车速还在上升
- 传动比会瞬间掉出所有档位的理论区间
- 导致卡档 1-2 秒

**解决方案**：
1. 检测换挡：`rpm < 上次转速 × 70%`
2. 换挡期间：保持原档位 10 帧（约 5 秒）

### 3.4 半离合蠕动检测（多维度组合）

**问题**：
- 半离合蠕动时（堵车、起步）
- rpm 可能很高（3000+），不是"rpm低"
- 特征是**传动比偏大**（离合器打滑）+ **传动比不稳定**

**解决方案**：三维度组合判断

```dart
bool _isCreepingState(speed, throttle, rpm, scaledRatio) {
  // 1. 速度范围过滤（必须条件）
  if (speed < 3 || speed > 15) return false;

  // 2. ratio 是否明显超过 1档
  bool ratioTooHigh = scaledRatio > maxTheoryRatio * 1.15;

  // 3. ratio 是否不稳定（核心特征：打滑导致波动）
  bool unstable = (scaledRatio - lastRatio).abs() / lastRatio > 0.05;

  // 4. 油门辅助条件
  bool lowThrottle = throttle < 20%;

  // 必须同时满足 3 个条件
  return ratioTooHigh && unstable && lowThrottle;
}
```

**判定条件**：
| 条件 | 说明 | 阈值 |
|------|------|------|
| `ratioTooHigh` | 传动比超过 1 档理论值 | ×1.15 |
| `unstable` | 传动比波动 | >5% |
| `lowThrottle` | 低油门辅助 | <20% |

**处理**：半离合状态时阈值放宽 1.5 倍

### 3.5 Scale Factor 校准

**问题**：
- 轮胎周长是理论值（1.97857 米）
- 实际有 ±5-8% 误差（胎压、载重、磨损、品牌差异）

**解决方案**：
1. 条件：速度 > 30 km/h，稳定 10 帧以上
2. 累加误差比值
3. 每 50 帧更新一次 scale
4. **档位变化检测**：如果在校准期间档位变化，重置采样

**档位变化检测**：
```dart
// 如果档位变化，重置采样，避免数据污染
if (_calibrationGear != 0 && _calibrationGear != bestGear) {
  _calibrationFrames = 0;
  _accumulatedError = 0;
}
_calibrationGear = bestGear;
```

**重要性**：如果不检测档位变化，采样会混合不同档位的理论值，导致校准结果被污染。

## 四、参数说明

### 4.1 车辆固有参数

| 参数 | 当前值 | 说明 | 是否需要调整 |
|------|--------|------|-------------|
| `primaryRatio` | 1.675 | 初级减速比 | 否 |
| `finalRatio` | 2.764 | 最终减速比 (47/17) | 改了牙盘需调整 |
| `tireCircumference` | 1.97857 | 轮胎周长 (米) | 改了轮胎需调整 |
| `internalGearRatios` | [3.071, 2.200, 1.700, 1.416, 1.230, 1.107] | 各档齿轮比 | 识别不准时调整 |

### 4.2 算法阈值参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `_thresholdBase` | 0.08 | 动态阈值基础值 |
| `_thresholdK` | 0.8 | 动态阈值系数 |
| `_stableFramesRequired` | 5 | 确认档位所需帧数 |

### 4.3 边界判定参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `_minSpeed` | 3 km/h | 低于此值视为空档 |
| `_minRpm` | 800 rpm | 低于此值视为空档 |

### 4.4 半离合检测参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `_creepMinSpeed` | 3 km/h | 半离合速度下限 |
| `_creepMaxSpeed` | 15 km/h | 半离合速度上限 |
| `_creepMaxThrottle` | 20% | 半离合油门上限 |
| `_creepThresholdMultiplier` | 1.5 | 半离合阈值放宽倍数 |

### 4.5 换挡检测参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `_shiftRpmDropRatio` | 0.7 | 换挡判定：转速下降到此比例 |
| `_shiftHoldFrames` | 10 | 换挡后保持帧数 |

### 4.6 校准参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `_calibrationMinSpeed` | 30 km/h | 校准生效最低速度 |
| `_calibrationStableFrames` | 10 | 校准所需稳定帧数 |
| `_calibrationErrorThreshold` | 0.10 | 校准误差阈值 |
| `_calibrationSampleFrames` | 50 | 校准采样帧数 |
| `_calibrationGear` | 0 | 当前校准档位（用于检测档位变化） |

```
总传动比 = primaryRatio × internalGearRatio × finalRatio
```

## 六、故障排查

### 6.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 4/5/6档来回跳动 | 高档位阈值太严格 | 调高阈值系数 k |
| 6档完全识别不出 | 齿轮比参数不准 | 校准 internalGearRatios[5] |
| 半离合时乱跳 | 半离合判断不准确 | 调整半离合检测阈值 |
| 换挡时卡顿 | 换挡保持时间太长 | 减少 _shiftHoldFrames |
| 一直显示空档 | 边界条件太严格 | 调低 _minSpeed 或 _minRpm |

### 6.2 手动校准方法

如果自动校准效果不好，可以手动记录各档位实际数据：

```
实测传动比 = (转速 × 60 × 轮胎周长) / (车速 × 1000)

实测齿轮比 = 实测传动比 ÷ (primaryRatio × finalRatio)
```

记录 6 档稳定行驶时的转速和车速，反推正确的齿轮比。
